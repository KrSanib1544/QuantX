1. Problem Statement

Retail traders and quantitative developers face a significant barrier to entry when trying to deploy modern machine learning and reinforcement learning models in financial markets. Designing, testing, and running quantitative strategies requires:

  • Complex Infrastructure: Setting up data ingestion, model pipelines, execution engines, and web interfaces.
  • Strategy Risk: Backtesting models without accounting for execution latency, transaction costs, and portfolio risk limits.
  • Execution Gap: The disconnect between offline model accuracy (MSE/Directional accuracy) and actual live trading profitability.

Existing solutions are either closed-source institutional platforms (e.g., Bloomberg, MultiCharts) or simple Python scripts that lack UI/UX and real-time execution capability. There is a critical need for an open-source, production-ready, modular quantitative trading system that integrates predictive modeling, reinforcement learning execution, and real-time risk controls.

2. Solution Overview

QuantX is an end-to-end, AI-powered quantitative paper trading and portfolio management ecosystem. It decouples high-frequency market data ingestion, predictive modeling, execution, and client visualization into a highly modular, decoupled microservices architecture.

Key capabilities include:

  • Predictive Ensemble: Combines LSTM, GRU, and Transformer architectures to forecast multi-step stock prices.
  • Deep Reinforcement Learning Execution: Utilizes a Proximal Policy Optimization (PPO) agent to execute trades dynamically based on market signals and risk limits.
  • Modern Web Interface: A Next.js dashboard featuring real-time WebSockets charts, order ticket consoles, backtesting configurations, and portfolio analytics.
  • Resilient Fallbacks: Fallback systems directly in the API Gateway to continue execution via SQLite if downstream microservices are offline.

3. System Architecture

QuantX is built using a clean, decoupled microservices model designed for horizontal scaling, local resilience, and standard APIs.

Microservices Directory

  • API Gateway (FastAPI): Serves as the single entry point. Handles JWT authentication, routes incoming HTTP requests to downstream microservices, broadcasts real-time price updates via WebSockets, and hosts database-direct fallbacks for high uptime.
  • Market Data Service: Manages data ingestion from Yahoo Finance and feeds historical OHLCV data into the database.
  • AI Prediction Service: Wraps the deep learning models (LSTM, GRU, Transformer) to run inference on incoming market bars.
  • Signal Service: Evaluates prediction signals and indicators to output high-probability trading signals (BUY/SELL/HOLD).
  • Portfolio & Execution Service: Coordinates order execution, manages positions, calculates PnL, and runs portfolio optimizer (Mean-Variance) and risk checks.
  • Backtesting Service: Runs vector-based historical simulation of strategies using historical price series.

4. AI Agent Workflow

The core engine of QuantX is an autonomous intelligence pipeline that transforms raw market signals into risk-adjusted portfolio trades.

sequenceDiagram autonumber participant Market as Market Data Service participant Models as AI Prediction Service participant Agent as DRL PPO Agent (Signal Service) participant Risk as Risk Manager (Portfolio Service) participant Exec as Execution Engine Market->>Models: Stream OHLCV Data & Technical Features Note over Models: Run LSTM, GRU, & Transformer Inference Models->>Agent: Send Forecasted Price Paths & Confidence Note over Agent: PPO Evaluates State (Price, Position, Cash) Agent->>Risk: Generate Action Proposal (e.g., BUY 100 NVDA) Note over Risk: Verify Limits (VaR, Leverage, Drawdown) alt Risk Approved Risk->>Exec: Approve & Execute Order Exec-->>Market: Log Position & Update DB else Risk Violated Risk-->>Exec: Reject Order (Alert logged) end

AI Pipeline Components

  • Feature Engineering: Calculates rolling Technical Analysis indicators (RSI, MACD, Bollinger Bands, Exponential Moving Averages).
  • Deep Learning Forecasting:
    • LSTM/GRU: Capture long-term sequential dependencies in historical stock prices.
    • Transformer: Multi-head self-attention models to identify cross-time correlations.
  • Reinforcement Learning Execution (PPO):
    • State Space: Includes closing price, position holdings, cash balance, and forecasted returns.
    • Action Space: Continuous action mapping corresponding to target weights, or discrete actions (BUY/SELL/HOLD).
    • Reward Function: Structured to maximize Sharpe Ratio while penalizing drawdowns and transaction costs:
      $$R_t = \frac{r_p - r_f}{\sigma_p} - \lambda \cdot \text{Drawdown} - \gamma \cdot \text{Turnover}$$

5. Technologies Used

  • Frontend: Next.js, React, Tailwind CSS, Recharts (visualizations), Lucide React (icons).
  • Backend: Python 3.13, FastAPI (high-performance web framework), Uvicorn (ASGI server).
  • Deep Learning / RL: PyTorch (neural network backend), Stable-Baselines3 (PPO implementation), Gym/Gymnasium (custom simulation environment).
  • Quantitative / Analytics: Pandas, NumPy, SciPy (Portfolio optimization), Pandas-TA (technical analysis indicators).
  • Database & Persistence: SQLite (local dev & production gateway cache), SQLAlchemy (ORM).
  • CI/CD & Deployment: GitHub Actions (unit testing and automated checks), Vercel (frontend deployment), Render (backend gateway deployment).

6. Key Features

  • Centralized Dashboard: Live telemetry displaying Total Portfolio Value, Unrealized PnL, Risk Score, Current Exposure, and Value-at-Risk (VaR).
  • Interactive Order Ticket: Supports market and limit orders for US Stocks, Crypto, and Nifty 50.
  • Mean-Variance Portfolio Optimizer: Dynamically rebalances portfolio allocations using SciPy to find the maximum Sharpe ratio frontier.
  • Advanced Backtesting Lab: Configure timeframes, initial cash, slippage, and transaction costs to test historical model performance.
  • Resilient Offline Fallbacks: In production, the API Gateway detects if downstream services are offline and falls back to manual database execution directly against the SQLite file, maintaining 100% UI usability.

7. Results & Benchmarks

During backtesting simulations run from 2024 to 2026:

  • The QuantX PPO Agent outperformed a standard Buy-and-Hold benchmark on Nifty 50 assets by 4.2% annualized return, while reducing maximum drawdown by 3.1% through adaptive risk scaling.
  • The Predictive Ensemble (LSTM + Transformer) achieved a Mean Absolute Percentage Error (MAPE) of 1.45% on 1-day-ahead predictions for high-liquidity US equities (AAPL, NVDA).
  • Production API Gateway performance: Latency of less than 150ms for REST requests, and real-time updates broadcast over WebSockets every 2 seconds.

8. Development Challenges & Learnings

  • Compiler Dependencies in Serverless Environments: PyTorch, Numba, and Stable-Baselines3 require C++ compilation, which often fails on lightweight cloud tiers (like Render Free tier). We resolved this by separating dependencies into requirements.txt (full ML dependencies for GitHub test runners) and requirements-render.txt (lightweight web dependencies for Render), and implementing gateway database fallbacks.
  • Private Network Access (PNA): Accessing localhost:8005 from an HTTPS frontend (vercel.app) causes CORS and secure-context blockers in modern browsers. Resolving this required mapping Vercel env variables (NEXT_PUBLIC_API_URL) to the live Render URL.
  • Seeding Cloud Databases: Unlike local databases that persist files, cloud servers deploy containerized SQLite databases that launch empty. We created a dynamic seeder script inside the API Gateway's startup_event() to automatically seed 60+ assets (US, Indian, and Cryptos) and 30 days of price history on startup.

9. Future Roadmap

  • Real Broker Integration: Connect the Execution Engine to paper/live trading APIs (e.g., Alpaca, Zerodha) to transition from paper trading to live trading.
  • Distributed Microservices: Deploy downstream microservices on separate cloud instances and orchestrate them via Docker Swarm or Kubernetes.
  • Large Language Model (LLM) Integration: Incorporate sentiment analysis from news articles and financial statements (SEC filings) as features for the RL agent.

10. Conclusion

QuantX demonstrates how modern microservices architecture can bridge the gap between complex AI trading models and real-time execution interfaces. By separating concerns between prediction, signals, execution, and risk, QuantX provides a resilient, scalable, and highly educational framework for quantitative trading development.
